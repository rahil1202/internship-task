## 1. üåê What is the DOM?

**DOM** stands for **Document Object Model**.
It‚Äôs a tree-like structure that the browser creates when it reads your HTML page. Every HTML element (like `<div>`, `<h1>`, `<p>`, `<input>`) becomes a **JavaScript object** inside this tree.

üëâ Think of the DOM as a **bridge between HTML and JavaScript**.
It lets you **access, read, and change HTML content, structure, and style** using JavaScript.

Example:

```html
<h1>Hello</h1>
```

The browser turns this into a DOM object like:

```
Document
‚îî‚îÄ html
   ‚îî‚îÄ body
      ‚îî‚îÄ h1  ‚Üí "Hello"
```

## 2. üîç Accessing Elements (Selecting Elements)

To manipulate anything, first you must **select** the element from the DOM.
There are multiple ways:

### a) `document.getElementById("id")`

* Finds **one element** by its `id`.
* Returns an **element object**.
* Fastest and most commonly used.

```js
const title = document.getElementById("title");
```

---

### b) `document.getElementsByClassName("className")`

* Finds **all elements** with a specific class.
* Returns a **live HTMLCollection** (updates automatically if DOM changes).

```js
const cards = document.getElementsByClassName("card");
```

---

### c) `document.getElementsByTagName("tagName")`

* Finds **all elements** with a specific tag (like `"p"`, `"div"`).
* Also returns a live HTMLCollection.

```js
const paragraphs = document.getElementsByTagName("p");
```

---

### d) `document.querySelector("cssSelector")`

* Finds the **first** element that matches a CSS selector.
* Most flexible and modern.

```js
const firstCard = document.querySelector(".card");
```

---

### e) `document.querySelectorAll("cssSelector")`

* Finds **all** matching elements.
* Returns a **NodeList** (not live).

```js
const allCards = document.querySelectorAll(".card");
```

‚úÖ **Tip:** `querySelector` and `querySelectorAll` are more modern and powerful because they use CSS-style selectors.

---

## 3. ‚úèÔ∏è Changing Content

Once you have selected an element, you can **change its text or HTML**.

* `.textContent` ‚Äì changes only the text inside an element.
* `.innerHTML` ‚Äì changes the HTML structure inside the element.

```js
const heading = document.getElementById("title");
heading.textContent = "Welcome Interns!";
heading.innerHTML = "<em>Welcome</em> Interns!";
```

---

## 4. üé® Changing Styles

You can dynamically change CSS styles from JavaScript using `.style`:

```js
heading.style.color = "blue";
heading.style.backgroundColor = "yellow";
heading.style.fontSize = "24px";
```

‚úÖ **Tip:** JavaScript uses **camelCase** for CSS properties (`background-color` ‚Üí `backgroundColor`).

---

## 5. üéõÔ∏è Working with Classes

It‚Äôs better to add/remove CSS **classes** instead of changing styles directly.
Use `.classList` methods:

* `.add("className")` ‚Äì add a class
* `.remove("className")` ‚Äì remove a class
* `.toggle("className")` ‚Äì add if not present, remove if present
* `.contains("className")` ‚Äì check if class exists

```js
heading.classList.add("highlight");
heading.classList.toggle("hidden");
```

---

## 6. üì• Reading and Using User Input

To read user input (like from a text field):

```html
<input type="text" id="name" />
```

```js
const name = document.getElementById("name").value;
```

`.value` is used for form elements like `<input>`, `<textarea>`, `<select>`.

---

## 7. ‚ûï Creating New Elements

You can create new HTML elements using `document.createElement()` and add them into the page with `.appendChild()` or `.append()`.

```js
const newItem = document.createElement("li");
newItem.textContent = "New Item";
document.getElementById("list").appendChild(newItem);
```

---

## 8. ‚ùå Removing Elements

To remove an element from the DOM, you can use `.remove()` directly:

```js
document.getElementById("title").remove();
```

Or remove a child from a parent:

```js
const list = document.getElementById("list");
list.removeChild(list.firstElementChild);
```

---

## 9. üîÑ Event Handling (Reacting to User Actions)

DOM interaction often involves **events** ‚Äî actions done by the user like clicking, typing, hovering, etc.

Two ways to handle events:

### a) Inline (basic)

```html
<button onclick="sayHello()">Click Me</button>
```

```js
function sayHello() {
  alert("Hello!");
}
```

### b) Using `.addEventListener()` (preferred)

```js
document.getElementById("btn").addEventListener("click", () => {
  alert("Hello!");
});
```

‚úÖ This method is cleaner, reusable, and allows multiple event listeners.

---

## 10. üîÅ Traversing the DOM (Parent, Child, Siblings)

Sometimes you need to move around the DOM tree:

```js
element.parentElement          // get parent
element.children              // get all child elements
element.nextElementSibling    // next element
element.previousElementSibling// previous element
```

## üßë‚Äçüíª Real-World Meaning of DOM Manipulation

üëâ DOM manipulation is **how we make websites interactive**.
Whenever a button changes text, a form shows a message, a list updates without refreshing, or a theme toggles ‚Äî that‚Äôs DOM manipulation in action.

It‚Äôs the **foundation** for modern web development and frameworks like React, Vue, and Angular.
They all manipulate the DOM ‚Äî they just automate it for you. So learning the basics by hand is essential.
